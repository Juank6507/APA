# Spec: Monitor Web Activo
Objetivo: Monitorizar página web con clicks automáticos, detectar cambios y alertar con sonido 30s.
Inputs: URL base, selectores CSS para botones "Aceptar" y "Continue/Continuar", selector elemento a monitorear.
Output esperado: Alarma sonora 30s cuando contenido cambia vs estado de referencia.
Criterio éxito: Detección cambio en <60s, alerta sonora funcional, ejecución cada 60s sin intervención.

## Flujo
1. Navegar a {URL} con Selenium (headless configurable)
2. Click en botón "Aceptar" → esperar carga → click en "Continue/Continuar"
3. Navegar a {URL}//#services, esperar <60s estabilización contenido
4. Extraer contenido elemento objetivo, normalizar, calcular hash SHA256
5. Comparar hash con último guardado en last_state.json (caché persistente)
6. Si hash diferente: reproducir alerta.wav 30s en hilo separado, actualizar caché, loguear
7. Esperar 60s, repetir desde paso 1 mientras active=True

## Validación estática (no ejecutar en sandbox)
- Código debe importar: selenium, webdriver-manager, playsound, hashlib, time, threading
- Debe tener funciones: navigate(), get_hash(), play_alert(), main_loop()
- Debe manejar excepciones sin propagar, logging estructurado, configuración vía .env
- Alerta: tono 880Hz, 30s, 70% volumen, patrón 0.5s on/off

## Configuración (.env)
URL_BASE=, SELECTOR_ACEPTAR=, SELECTOR_CONTINUE=, SELECTOR_MONITOREO=, INTERVALO=60, ALERTA_DURACION=30

## Output archivos
monitor.py (principal), config.py (carga .env), alert.wav (sonido), last_state.json (caché hash), requirements.txt (selenium, webdriver-manager, playsound)

## Notas
- Normalizar contenido antes de hash: eliminar espacios extra, lowercase, strip
- Reintentos máx 3 en navegación, logging de cada intento
- Ctrl+C detiene limpiamente, guarda estado actual
- Si API/URL falla, loguear warning y continuar siguiente ciclo
