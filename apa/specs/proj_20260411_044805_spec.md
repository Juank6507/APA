# Spec: Monitor Web Activo con Interacción y Alerta por Cambio de Estado
Modo: generación de aplicación
Proyecto: WebMonitor APA

## Objetivo
Crear una aplicación Python que monitoree activamente una página web requiriendo interacción automatizada previa (dos clicks secuenciales), detecte cambios en el estado del contenido y emita una alerta sonora de 30 segundos cuando se detecte una variación respecto al estado de referencia.

## Inputs disponibles
- apa/core/orchestrator.py con clase Orchestrator y método run(spec_path, on_progress)
- apa/mcp/server.py con NASConnector para ejecución en sandbox
- apa/core/router.py con call_llm para generación de código
- Librerías estándar de Python: selenium, webdriver-manager, playsound, time, hashlib, requests, threading
- apa/agents/generator.py y corrector.py para generación y corrección autónoma

## Flujo de la aplicación a generar

### 1. Configuración inicial (archivo config.py)
- URL base del servidor a monitorear (configurable vía .env)
- URL completa final: {url_base}//#services
- Selectores CSS/XPath para los botones:
  - Botón 1: "Aceptar" (página de bienvenida)
  - Botón 2: "Continue" o "Continuar" (página intermedia)
- Selector del elemento a monitorear en la página final (configurable)
- Intervalo de monitoreo: 60 segundos
- Duración de alerta sonora: 30 segundos

### 2. Módulo de navegación automatizada (navigator.py)
- Usar Selenium con WebDriver (Chrome headless configurable)
- Función navigate_to_services(url_base, selectors) que:
  - Abre el navegador en modo headless o visible (configurable)
  - Navega a url_base
  - Espera y hace click en botón "Aceptar" (timeout 10s)
  - Espera carga y hace click en botón "Continue/Continuar" (timeout 10s)
  - Navega a {url_base}//#services
  - Espera hasta 60 segundos a que el contenido se estabilice
  - Retorna el HTML o texto del elemento objetivo normalizado

### 3. Módulo de detección de cambios (detector.py)
- Función get_content_hash(element_selector) que:
  - Extrae el contenido del elemento objetivo
  - Normaliza: elimina espacios extra, convierte a minúsculas si aplica
  - Genera hash SHA256 del contenido normalizado
  - Retorna el hash para comparación
- Función has_changed(previous_hash, current_hash) que compara hashes
- Almacenamiento del último estado conocido en archivo local last_state.json con timestamp

### 4. Módulo de alerta (alerter.py)
- Función play_alert(duration_seconds=30) que:
  - Reproduce un sonido de alerta (archivo .wav incluido o generado con wave)
  - Se ejecuta en hilo separado para no bloquear el monitoreo
  - Se detiene automáticamente tras 30 segundos
- Función log_alert(timestamp, change_details) para registro en consola/archivo

### 5. Módulo principal (monitor.py)
- Bucle principal que se ejecuta mientras active=True:
while active:
      navigate_to_services(...)
      current_hash = get_content_hash(...)
      if has_changed(saved_hash, current_hash):
          play_alert(30)
          log_alert(...)
          save_state(current_hash)  # Actualizar estado de referencia
      sleep(60)  # Esperar 1 minuto para siguiente ciclo- Manejo de señales (Ctrl+C) para detener limpiamente
- Logging estructurado de cada ciclo: timestamp, hash anterior, hash actual, cambio detectado (Sí/No)
- Archivo de configuración .env para URL y selectores

### 6. Interfaz de control mínima (opcional, cli.py)
- Opción CLI: python monitor.py --start, --stop, --status
- Muestra estado actual: activo/inactivo, último hash, último cambio detectado

## Output esperado
- Aplicación funcional que se ejecuta con python monitor.py --start
- Archivo config.py editable para personalizar URL y selectores
- Archivo de sonido alert.wav incluido (o generado con wave)
- Archivo last_state.json para persistir el estado de referencia entre ejecuciones
- Logs en consola y archivo monitor.log con formato:
  [2026-04-11 14:30:00] Ciclo #45 | Hash: a1b2c3... | Cambio: NO
  [2026-04-11 14:31:00] Ciclo #46 | Hash: x9y8z7... | Cambio: SÍ → ALERTA ACTIVADA (30s)
## Criterio de éxito
1. La aplicación navega automáticamente y realiza los dos clicks requeridos sin intervención manual
2. Accede correctamente a {url_base}//#services y espera la carga del contenido
3. Detecta cambios en el elemento objetivo comparando hashes del contenido normalizado
4. Emite una alerta sonora de exactamente 30 segundos cuando se detecta un cambio
5. Se ejecuta en bucle cada 60 segundos mientras esté activa
6. Permite detenerse limpiamente con Ctrl+C o comando --stop
7. El código generado pasa validación estática (sintaxis Python válida, imports correctos)
8. No consume recursos excesivos: uso de CPU <10% en idle, memoria <200MB

## Notas adicionales
- Usar webdriver-manager para gestión automática de drivers de Selenium
- Incluir requirements.txt con dependencias: selenium, webdriver-manager, playsound
- El sonido de alerta puede ser un tono simple generado con la librería wave o un archivo .wav predefinido
- Normalizar el contenido antes de hashear para evitar falsos positivos por cambios cosméticos (espacios, saltos de línea)
- Implementar reintentos (máx. 3) en caso de fallo en la navegación o carga de página, con logging de cada intento
- El estado de referencia (last_state.json) debe incluir timestamp para saber cuándo se estableció por última vez

2. navigator.py ← navegación automatizada con Selenium
3. detector.py ← detección de cambios por hash
4. alerter.py ← sistema de alerta sonora
5. config.py ← configuración cargada desde .env
6. requirements.txt ← dependencias del proyecto
7. alert.wav ← sonido de alerta (30 segundos, tono simple)

## Especificación de la alerta
- Formato: WAV, 44.1kHz, mono, 16-bit
- Duración: exactamente 30 segundos
- Contenido: tono de 880Hz (La5) con patrón intermitente (0.5s on, 0.5s off) para ser audible pero no intrusivo
- Volumen: 70% del máximo para no saturar

## Especificación de normalización de contenido
Antes de calcular el hash, el contenido debe:
1. Eliminar etiquetas HTML si el selector retorna HTML
2. Reemplazar múltiples espacios/saltos por un solo espacio
3. Convertir a minúsculas (opcional, configurable)
4. Eliminar caracteres no imprimibles excepto espacios
Esto asegura que cambios de formato no disparen falsas alarmas.

## Manejo de errores
- Si la navegación falla 3 veces consecutivas: loguear error y continuar con siguiente ciclo
- Si el elemento objetivo no se encuentra: loguear warning y usar hash vacío como referencia temporal
- Si la alerta sonora falla: loguear error pero no interrumpir el monitoreo
- Nunca lanzar excepción al caller — todo capturado y logueado

## Pruebas de validación
1. Ejecutar con URL de prueba y verificar que realiza los dos clicks correctamente
2. Simular cambio de contenido y verificar que la alerta se activa
3. Verificar que el hash se actualiza tras detectar cambio (no alerta repetida)
4. Verificar que Ctrl+C detiene limpiamente el proceso
5. Verificar que last_state.json se crea y actualiza correctamente
