⚠️ Consideraciones técnicas para cuando el APA la procese:

1. Selenium requiere Chrome/Chromium instalado en la máquina donde se ejecute.
   Si el APA intenta ejecutar en el sandbox del NAS, fallará porque Selenium necesita:
   - Un navegador real o configuración headless con drivers
   - Dependencias gráficas que el contenedor Docker del NAS no tiene por defecto

2. Solución recomendada:
   - Añadir a la spec esta línea en "Validación estática":
     "Esta aplicación se ejecuta LOCALMENTE en Windows, no en sandbox NAS"
   - O modificar el generator.py para detectar tareas con "selenium" y forzar validación estática + entrega de archivos sin ejecución

3. El archivo alert.wav:
   - El APA puede generarlo con la librería `wave` + `struct` (tono 880Hz, 30s, patrón 0.5s on/off)
   - O incluirlo como archivo binario predefinido en el output

4. Dependencias:
   - requirements.txt debe incluir: selenium, webdriver-manager, playsound
   - webdriver-manager descargará automáticamente el ChromeDriver compatible
